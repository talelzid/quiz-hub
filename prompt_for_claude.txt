Les métriques du logiciel

Une métrique du logiciel sert à quantifier une propriété du monde logiciel.

Exemple: nombre d’erreurs dans un programme: permet de quantifier la “qualité”

du programme.
L’utilisation des métriques est une nécessité pour un chef de projet logiciel et pour

l'ensemble des membres de son équipe.
L'utilisation des métriques doit être prévue à l’avance.

Les métriques ont plusieurs rôles en gestion de projet :

Savoir où on en est (temps, coût, risque).

Evaluer le travail effectué (estimation d’effort, pertinence, quantité, productivité).

Evaluer le produit (pertinence, qualité, mesure de complexité).

Détecter des problèmes imminents ou potentiels.

Faire des prévisions.

L’analyse statique du code
18
C’est une technique de vérification statique.

Elle consiste à examiner le code de manière automatisée à l’aide d’un outil.

Le but étant d’identifier le maximum de défauts et de vulnérabilités connues afin de

les corriger.
Cette technique n’est pas capable d’identifier toutes les failles en parcourant le code

source.
Elle constitue donc est un complément précieux aux méthodes classiques de révision

de code (Revues).
La démarche idéale consiste à commencer par une analyse statique du code (à l’aide

d’un outil) et de continuer avec une revue de code.
